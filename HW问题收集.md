## HW 问题收集

列举在HW 1、HW1.1过程里，你所遇到的2\~3个通过自己学习已经解决的问题，和2~3个尚未解决的问题与挑战

### 已解决

1. 不熟悉项目结构
   1. **上下文**：一开始不知道游戏启动流程、数据流向、哪些组件依赖哪些 store
   2. **解决手段**：通过阅读源码和 CA 引导，理解了从 Welcome.svelte → startNew() → grid.generate() → UI 渲染的原始完整链路
2. 不了解 Svelte 运行机制
   1. **上下文**：不知道 UI 为什么能响应式更新，Svelte 怎么知道数据变了
   2. **解决手段**：通过自己提问和 CA 解答，理解了 Svelte 的响应式基于"赋值触发订阅通知"——`state.set({...})` 重新赋值触发通知，`$store` 能消费是因为 store 有 subscribe 方法

### 未解决

1. 不确定是否要保留原项目逻辑
   1. **上下文**：在接入领域对象时，不确定哪些旧 store（grid、userGrid、invalidCells）要保留，哪些要弃用
   2. **尝试解决手段**：CA 解答后理解了只要 UI 改用 gameStore，旧的 store 可以弃用，但具体哪些要彻底删除还不确定
2. 不理解面向对象下应该使用多少个对象才合适
   1. **上下文**：不知道除了 Sudoku 和 Game， 是否应该建立更多对象，划分更多职责
   2. **尝试解决手段**：目前采用两个对象的设计（Sudoku 负责棋盘数据，Game 负责历史和流程），但不确定这是不是最优解
